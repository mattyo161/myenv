# Configuration reference

Every option that controls what this playbook does, in one place. Most live in
`group_vars/`; a few are role defaults or runtime controls (tags, extras
toggles, environment variables).

**Where settings live**

| File | Scope | Committed? | Purpose |
| --- | --- | --- | --- |
| `group_vars/all.yml` | Everyone | Yes | Shared package/tool inventories. |
| `group_vars/local.yml` | This machine | No (git-ignored) | Per-user identity; generated by `bootstrap.sh`. |
| `roles/<role>/defaults/main.yml` | Per role | Yes | Role behavior defaults; override in `group_vars`/`host_vars`. |

Precedence (later wins): role `defaults` → `group_vars/all.yml` → `group_vars/local.yml` → `-e` on the command line.

---

## Identity (required)

Set in `group_vars/local.yml` (created by `bootstrap.sh`). The play **aborts**
in `pre_tasks` if any is missing or still the placeholder value.

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `git_user_name` | string | `Your Name` (placeholder) | Name written to git config and chezmoi template data. |
| `git_user_email` | string | `you@example.com` (placeholder) | Email for git config and chezmoi template data. |
| `github_user` | string | `your-gh-user` (placeholder) | GitHub username; exposed to dotfiles as `github_user`. |

## Package & tool inventories

Set in `group_vars/all.yml`. These are plain lists/maps — add or remove entries
to change what gets installed. Comment a line out to skip it.

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `homebrew_taps` | list[string] | 3 taps (`cloudgraphdev/tap`, `siderolabs/tap`, `supabase/tap`) | Third-party Homebrew taps to add. Trusting the tap covers all its formulae/casks. |
| `homebrew_formulae` | list[string] | ~20 CLI tools | Homebrew **formulae** (CLI tools) to install. |
| `homebrew_casks_common` | list[string] | ~12 apps | Homebrew **casks** (GUI apps) installed for everyone. Includes `visual-studio-code` so the `code` CLI exists for the vscode role. |
| `homebrew_casks_personal` | list[string] | `google-chrome`, `logseq`, `obsidian` | Casks installed only when the `personal` tag is **not** skipped. |
| `mise_tools` | map[string→string] | ~25 tools | mise-managed runtimes/tools as `tool: version`. Use `latest` or pin (e.g. `node: "20.19.2"`). |
| `mise_idiomatic_version_file_enable_tools` | list[string] | `python`, `node`, `terraform` | Tools whose idiomatic version file (e.g. `.python-version`) mise should honor. |
| `vscode_extensions` | list[string] | ~10 extensions | VS Code extension IDs installed via the `code` CLI. |

## Role behavior

| Option | Type | Default | Set in | Description |
| --- | --- | --- | --- | --- |
| `dotfiles_enforce` | bool | `true` | `roles/dotfiles/defaults/main.yml` | `true`: `chezmoi apply --force` — repo wins, local edits to managed dotfiles are overwritten silently. `false`: `chezmoi apply --no-tty --keep-going` — never tries to open a TTY, applies non-conflicting files, and leaves files changed outside chezmoi untouched (a `[WARNING]` lists them). See [decision 0001](decisions/0001-chezmoi-conflict-handling.md). |
| `homebrew_prefix` | string (path) | `/opt/homebrew` | unset (override only) | Homebrew install prefix. Defaulted inline, not declared. Override to `/usr/local` for Intel Macs so the dotfiles role finds `chezmoi`. |

## Runtime controls (command line)

Not variables, but the main levers for *what runs*.

### Tags

Each role is tagged with its own name; `personal` marks personal-only apps.

| Tag | Selects |
| --- | --- |
| `xcode`, `homebrew`, `mise`, `vscode`, `extras`, `dotfiles` | The matching role. |
| `personal` | Personal-only steps (e.g. `homebrew_casks_personal`). |
| `always` | Identity/OS pre-checks; never skipped. |

```sh
ansible-playbook site.yml                      # everything
ansible-playbook site.yml --tags mise          # one role only
ansible-playbook site.yml --skip-tags personal # omit personal-only apps
ansible-playbook site.yml --check --diff       # dry run
```

### Extras (opt-in add-ons)

The `extras` role runs every `*.yml` in `roles/extras/tasks/enabled/`. Toggle an
extra by moving its file between `enabled/` and `disabled/` — nothing in
`disabled/` ever runs.

| State | Files (as of writing) |
| --- | --- |
| Enabled | `10-aws`, `20-azure`, `30-gcp`, `40-kubernetes`, `50-iac`, `60-containers-security`, `80-apps-core`, `82-apps-editors`, `83-apps-productivity` |
| Disabled | `70-local-k8s`, `81-apps-infra`, `90-mac-app-store` |

### Environment variables

| Variable | Used by | Description |
| --- | --- | --- |
| `HOMEBREW_REQUIRE_TAP_TRUST` | Homebrew | When set, Homebrew only loads formulae/casks from trusted taps. The homebrew role runs `brew trust` on each tap in `homebrew_taps` so they load. |

## Ansible engine settings (`ansible.cfg`)

Defaults that affect execution, not what gets installed.

| Key | Value | Description |
| --- | --- | --- |
| `inventory` | `inventory.ini` | Inventory file. |
| `roles_path` | `roles` | Where roles are found. |
| `stdout_callback` | `default` | Console output format driver. |
| `callback_plugins` | `callback_plugins` | Local callback plugin directory. |
| `callbacks_enabled` | `warn_facts` | Enables the [`warn_facts`](../callback_plugins/warn_facts.py) callback that turns a `callback_warnings` fact into a real `[WARNING]`. |
| `result_format` | `yaml` | Task result rendering. |
| `host_key_checking` | `False` | Skip SSH host-key prompts. |
| `interpreter_python` | `auto_silent` | Auto-discover the Python interpreter quietly. |
