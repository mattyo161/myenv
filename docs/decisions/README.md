# Design decisions

Short records of notable design/architecture choices in this repo — what we
decided, and **why** — so future readers (and our future selves) don't have to
reverse-engineer the reasoning or relitigate settled trade-offs.

Each decision is its own numbered Markdown file. Keep them brief: context, the
decision, alternatives considered, and consequences. Add a new one whenever a
change involves a non-obvious trade-off.
